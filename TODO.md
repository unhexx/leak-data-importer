# TODO: leak-data-importer — Полная дорожная карта

**Проект:** Инструмент безопасного импорта, нормализации и анализа утечек и чувствительных данных.  
**Текущий статус:** Alpha (v0.1.0)  
**Дата обновления:** 2026-05-28  
**Основной фокус:** Разработка устойчивого пайплайна обработки OSINT/утечек с поддержкой графового анализа.

---

## 1. Цель проекта (высокоуровневая)

Создать production-ready инструмент, который позволяет:

- Безопасно импортировать данные из различных форматов утечек
- Качественно нормализовывать русскоязычные данные (ФИО, телефоны, паспорта, СНИЛС, адреса и т.д.)
- Строить граф связей между сущностями
- Выполнять дедупликацию и entity resolution
- Экспортировать данные в удобные для анализа форматы (Neo4j, CSV, JSON Lines, Parquet и др.)
- Предоставлять удобный CLI и программный интерфейс

## 3. Технический долг и риски

- Жёсткая привязка к Windows PowerShell в части tooling (нужна кросс-платформенность)
- Отсутствие полноценной работы с PII (риск compliance)
- Недостаточное количество реальных тестов на "грязных" данных
- Нет стратегии версионирования схемы данных

---

## 4. Приоритетные направления на ближайшие спринты

1. **Граф + Neo4j** — довести до рабочего состояния (самая ценная фича)
2. **База данных** — SQLAlchemy 2.0 + миграции
3. **Дедупликация и linking** — ключевая аналитическая возможность
4. **Качественные тесты** на реальных примерах
5. **Улучшение нормализации** (особенно документов и адресов)
6. **Развитие агентного цикла разработки** (чтобы проект мог развиваться автономно)

## 5. Как обновлять этот файл

- Обновляется **Reviewer** в конце каждого агентного цикла
- Добавляются новые задачи только после обсуждения с Orchestrator
- Все задачи должны быть декомпозированы до уровня, когда их можно взять в работу за 1–3 дня
