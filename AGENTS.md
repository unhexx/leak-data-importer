# AGENTS.md — Правила работы с циклами разработки

> Переносимый набор правил для работы в Grok CLI, Blackbox, Cursor, Continue и других средах (оптимизировано под сильные модели текущего поколения на проекте leak-data-importer).

## Основной цикл (Solver Loop) — для agentless-режима

1. **Inspect** — сначала полностью прочитай релевантный код (parsers/normalization, importers, graph, тесты, фикстуры). Не начинай править, пока не поймёшь «spine» системы.
2. **Define success** — чётко опиши measurable исход (тест прошёл на фикстуре, нормализация СНИЛС/телефона дала ожидаемый результат, графовая связь создана корректно и т.д.).
3. **Smallest vertical slice** — реализуй минимальный end-to-end кусок, который доказывает, что основная идея работает.
4. **Proportional verification** — проверь на поверхности пользователя (pytest + ручная симуляция на фикстурах + логи). Фиксируй доказательства.
5. **Reflect** — что сработало, что нет, нужно ли откатить через git. Только после этого расширяй scope.

Максимум 3 tool-вызова без рефлексии внутри кластера.

## Два режима работы

- **Agentless (рекомендуется для большинства задач в этом CLI / прямой работе)**: одна сильная модель следует Solver Loop напрямую. Лёгкий оверхед, быстрые итерации. Поддержка в `agentless_loop/`. Используй по умолчанию в Grok, Blackbox (прямой чат), Cursor.
- **Agentic (полноценный)**: многоролевая машина (Orchestrator → Coder → Tester → Debugger → Reviewer) со строгими JSON-хandoff. Больше проверок, больше артефактов (handoff_*.json и т.д.). Шаблон в `agentic_loop_template/`. Подключай при высокой сложности, Phase 5 экспортерах, рефакторингах схемы или когда нужна жёсткая разделённая ответственность.

## Важные технические рекомендации

- При работе с моделями, поддерживающими reasoning traces, сохраняй их в истории контекста — это значительно повышает качество длинных итераций.
- Для творческих ролей (реализация, отладка парсеров) полезны более высокие значения temperature. Для планирования, тестирования и ревью — низкие.
- Контекст больших моделей позволяет работать с репозиторием. Агрессивно суммируй историю, оставляй только релевантные факты, метрики прошлых попыток и ключевые решения.
- Всегда работай только с фикстурами и синтетическими данными. Реальные файлы утечек в `data/raw/` — только для анализа, никогда не коммить и не выводи.

## Постоянные правила проекта (leak-data-importer)

- Все коммиты, комментарии в коде и докстринги — только на естественном русском, стиль реального mid/senior разработчика (см. `agentic_loop_template/DEVELOPMENT_STANDARDS.md`).
- **Полный запрет** упоминания AI, LLM, агент, нейросеть, MiniMax, Grok, Claude и т.п. в коммитах и комментариях к коду.
- Любая работа с Python — только внутри `.venv` (используй `scripts/setup.ps1` или эквивалент из agentic_loop_template).
- Весь импорт, нормализация, разрешение сущностей и построение графа — строго детерминированные. LLM никогда не вызывается внутри основного pipeline.
- PII и безопасность данных — критичны. Любые изменения в redaction/normalization требуют особой внимательности.
